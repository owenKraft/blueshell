  <p>
    Starting in Windows PowerShell 3.0, this cmdlet has been superseded by
    Get-CimInstancehttp://technet.microsoft.com/library/jj590758(v=wps.630).aspx.
  </p>
  <p>
    The Get-WmiObject cmdlet gets instances of Windows Management
    Instrumentation (WMI) classes or information about the available WMI
    classes. To specify a remote computer, use the ComputerName parameter. If
    the List parameter is specified, the cmdlet gets information about the WMI
    classes that are available in a specified namespace. If the Query parameter
    is specified, the cmdlet runs a WMI query language (WQL) statement.
  </p>
  <p>
    This cmdlet does not use Windows PowerShell remoting to perform remote
    operations. You can use the ComputerName parameter of the Get-WmiObject
    cmdlet even if your computer does not meet the requirements for Windows
    PowerShellremoting or is not configured for remoting in Windows PowerShell.
  </p>
  <p>
    Beginning in Windows PowerShell 3.0, the __Server property of the object
    that Get-WmiObject returns has a PSComputerName alias. This makes it easier
    to include the source computer name in output and reports.
  </p>
  <h5>Parameters</h5>
  <p><code id="Amended">-Amended [&lt;SwitchParameter&gt;]</code></p>
  <ul>
    <li><em>Default value is</em> <strong>False</strong></li>
    <li><em>Accepts pipeline input </em><strong>False</strong></li>
  </ul>
  <div class="indent">
    <p>
      Indicates whether the objects that are returned from WMI should contain
      amended information. Typically, amended information is localizable
      information, such as object and property descriptions, that is attached to
      the WMI object.
    </p>
  </div>

  <p><code id="AsJob">-AsJob [&lt;SwitchParameter&gt;]</code></p>
  <ul>
    <li><em>Default value is</em> <strong>False</strong></li>
    <li><em>Accepts pipeline input </em><strong>False</strong></li>
  </ul>
  <div class="indent">
    <p>
      Indicates that this cmdlet runs the command as a background job. Use this
      parameter to run commands that take a long time to finish.
    </p>
    <p>
      When you use the AsJob parameter, the command returns an object that
      represents the background job and then displays the command prompt. You
      can continue to work in the session while the job finishes. If
      Get-WmiObject is used on a remote computer, the job is created on the
      local computer, and the results from remote computers are automatically
      returned to the local computer. To manage the job, use the cmdlets that
      contain the Job cmdlets. To get the job results, use the Receive-Job
      cmdlet.
    </p>
    <p>
      To use this parameter with remote computers, the local and remote
      computers must be configured for remoting. Additionally, you must start
      Windows PowerShell by using the Run as administrator option in Windows
      Vista and later versions of Windows.
    </p>
    <p></p>
  </div>

  <p>
    <code id="Authentication">-Authentication &lt;AuthenticationLevel&gt;</code>
  </p>
  <ul>
    <li><em>Default value is</em> <strong>None</strong></li>
    <li><em>Accepts pipeline input </em><strong>False</strong></li>
  </ul>
  <div class="indent">
    <p>
      Specifies the authentication level to be used with the WMI connection. The
      acceptable values for this parameter are:
    </p>
    <p>-1: Unchanged</p>
    <p>0: Default</p>
    <p>1: None (No authentication in performed.)</p>
    <p>
      2: Connect (Authentication is performed only when the client establishes a
      relationship with the application.)
    </p>
    <p>
      3: Call (Authentication is performed only at the beginning of each call
      when the application receives the request.)
    </p>
    <p>
      4: Packet (Authentication is performed on all the data that is received
      from the client.)
    </p>
    <p>
      5: PacketIntegrity (All the data that is transferred between the client
      and the application is authenticated and verified.)
    </p>
    <p>
      6: PacketPrivacy (The properties of the other authentication levels are
      used, and all the data is encrypted.)
    </p>
  </div>

  <p><code id="Authority">-Authority &lt;String&gt;</code></p>
  <ul>
    <li><em>Default value is</em> <strong>None</strong></li>
    <li><em>Accepts pipeline input </em><strong>False</strong></li>
  </ul>
  <div class="indent">
    <p>
      Specifies the authority to use to authenticate the WMI connection. You can
      specify standard NTLM or Kerberos authentication. To use NTLM, set the
      authority setting to ntlmdomain:&lt;DomainName&gt;, where
      &lt;DomainName&gt; identifies a valid NTLM domain name. To use Kerberos,
      specify kerberos:&lt;DomainName&gt;\&lt;ServerName&gt;". You cannot
      include the authority setting when you connect to the local computer.
    </p>
  </div>

  <p><code id="Class">-Class &lt;String&gt;</code></p>
  <ul>
    <li><em>This value is</em> <span style="color: #333">required</span></li>
    <li><em>Default value is</em> <strong>None</strong></li>
    <li><em>Accepts pipeline input </em><strong>False</strong></li>
  </ul>
  <div class="indent">
    <p>
      Specifies the name of a WMI class. When this parameter is used, the cmdlet
      retrieves instances of the WMI class.
    </p>
  </div>

  <p><code id="ComputerName">-ComputerName &lt;String[]&gt;</code></p>
  <ul>
    <li><em>Default value is</em> <strong>None</strong></li>
    <li><em>Accepts pipeline input </em><strong>False</strong></li>
  </ul>
  <div class="indent">
    <p>
      Specifies the target computer for the management operation. Enter a fully
      qualified domain name (FQDN), a NetBIOS name, or an IP address. When the
      remote computer is in a different domain than the local computer, the
      fully qualified domain name is required.
    </p>
    <p>
      The default is the local computer. To specify the local computer, such as
      in a list of computer names, use "localhost", the local computer name, or
      a dot (.).
    </p>
    <p>
      This parameter does not rely on Windows PowerShell remoting, which uses
      WS-Management. You can use the ComputerName parameter of Get-WmiObject
      even if your computer is not configured to run WS-Management remote
      commands.
    </p>
  </div>

  <p><code id="Credential">-Credential &lt;PSCredential&gt;</code></p>
  <ul>
    <li><em>Default value is</em> <strong>None</strong></li>
    <li><em>Accepts pipeline input </em><strong>False</strong></li>
  </ul>
  <div class="indent">
    <p>
      Specifies a user account that has permission to perform this action. The
      default is the current user. Type a user name, such as User01,
      Domain01\User01, or <a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="194c6a7c6b595a76776d766a76377a7674">[email&#160;protected]</a>. Or, enter a PSCredential object,
      such as an object that is returned by the Get-Credential cmdlet. When you
      type a user name, you are prompted for a password.
    </p>
  </div>

    <li><em>Accepts pipeline input </em><strong>False</strong></li>
  </ul>
  <div class="indent">
    <p>
      Specifies whether direct access to the WMI provider is requested for the
      specified class without any regard to its base class or to its derived
      classes.
    </p>
  </div>

    <code id="EnableAllPrivileges"
      >-EnableAllPrivileges [&lt;SwitchParameter&gt;]</code
    >
  </p>
  <ul>
    <li><em>Default value is</em> <strong>False</strong></li>
    <li><em>Accepts pipeline input </em><strong>False</strong></li>

    </p>

  <p><code id="Filter">-Filter &lt;String&gt;</code></p>
  <ul>
    <li><em>Default value is</em> <strong>None</strong></li>
    <li><em>Accepts pipeline input </em><strong>False</strong></li>
  </ul>
  <div class="indent">
    <p>
      Specifies a Where clause to use as a filter. Uses the syntax of the WMI
      Query Language (WQL).
    </p>
    <p>
      Important: Do not include the Where keyword in the value of the parameter.
      For example, the following commands return only the logical disks that

      using the Where keyword.
    </p>
    <p>`Get-WmiObject Win32_LogicalDisk -filter "DeviceID = 'c:' "`</p>
    <p>`Get-WmiObject win32_service -filter "name='WinRM'"`</p>
  </div>

  <p>
    <code id="Impersonation">-Impersonation &lt;ImpersonationLevel&gt;</code>
  </p>
  <ul>
    <li><em>Default value is</em> <strong>None</strong></li>
    <li><em>Accepts pipeline input </em><strong>False</strong></li>
  </ul>
  <div class="indent">
    <p>Specifies the impersonation level to use.</p>
    <p>The acceptable values for this parameter are:</p>
    <p>
      0: Default. Reads the local registry for the default impersonation level ,
      which is usually set to "3: Impersonate".
    </p>
    <p>1: Anonymous. Hides the credentials of the caller.</p>
    <p>2: Identify. Allows objects to query the credentials of the caller.</p>
    <p>3: Impersonate. Allows objects to use the credentials of the caller.</p>
    <p>
      4: Delegate. Allows objects to permit other objects to use the credentials
      of the caller.
    </p>
  </div>

  <p><code id="List">-List [&lt;SwitchParameter&gt;]</code></p>
  <ul>
    <li><em>Default value is</em> <strong>False</strong></li>
    <li><em>Accepts pipeline input </em><strong>False</strong></li>
  </ul>
  <div class="indent">
    <p>
      Gets the names of the WMI classes in the WMI repository namespace that is
      specified by the Namespace parameter.
    </p>
    <p>
      If you specify the List parameter, but not the Namespace parameter,
      Get-WmiObject uses the Root\Cimv2 namespace by default. This cmdlet does
      not use the Default Namespace registry entry in the
      HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\WBEM\Scripting registry key to
      determine the default namespace.
    </p>
  </div>

</code></pre>

          <p>This command get the processes on the local computer.</p>
        </div>
      </li>
      <li>
        <em>Gets services on a remote computer:</em>
        <pre><code>PS C:\&gt; <span class="hljs-built_in">Get-WmiObject</span> -Class Win32_Service -ComputerName <span class="hljs-number">127.0</span>.<span class="hljs-number">0.1</span>
</code></pre>
        <div class="indent">
          <p>
            This command gets the services on a remote computer. It uses the
            ComputerName parameter to specify the Internet Protocol (IP)

            of the Administrators group on the remote computer.
          </p>
        </div>
      </li>
      <li>
        <em
          >Get WMI classes in the root or default namespace of the local
          computer:</em
        >
        <pre><code>PS C:\&gt; <span class="hljs-built_in">Get-WmiObject</span> -Namespace <span class="hljs-string">"root/default"</span> -List
</code></pre>
        <div class="indent">
          <p>
            This command gets the WMI classes in the root or default namespace
            of the local computer.
          </p>
        </div>
      </li>
      <li>
        <em>Get a named service on multiple computers:</em>
        <pre><code>PS C:\&gt; <span class="hljs-built_in">Get-WmiObject</span> -Query <span class="hljs-string">"select * from win32_service where name='WinRM'"</span> -ComputerName Server01, Server02 | <span class="hljs-built_in">Format-List</span> -Property PSComputerName, Name, ExitCode, Name, ProcessID, StartMode, State, Status

        <div class="indent">
          <p>
            This command gets the WinRM service on the computers that are
            specified by the value of the ComputerName parameter.
          </p>
          <p>
            A pipeline operator (|) sends the output to the Format-List cmdlet,
            which adds the PSComputerName property to the default output. This
            makes it easy to see the computer on which the service resides.
            PSComputerName is an alias of the __Server property of the objects
            that Get-WmiObject returns. This alias is introduced in Windows
            PowerShell 3.0.
          </p>
        </div>
      </li>
      <li>
        <em>Stop a service on a remote computer:</em>
        <pre><code>PS C:\&gt; (<span class="hljs-built_in">Get-WmiObject</span> -Class Win32_Service -Filter <span class="hljs-string">"name='WinRM'"</span> -ComputerName Server01).StopService()
</code></pre>
        <div class="indent">
          <p>
            This command stops the WinRM service on the Server01 remote
            computer. The command uses a Get-WmiObject command to get the WinRM
            service on Server01. Then, it invokes the StopService method of the
            Win32_Service WMI class on the object that the Get-WmiObject command
            returns.
          </p>
          <p>
            This command is an alternative to using the Stop-Service cmdlet.
          </p>
        </div>
      </li>

 Only some of the properties of each WMI class are displayed by default. The set of properties that is
